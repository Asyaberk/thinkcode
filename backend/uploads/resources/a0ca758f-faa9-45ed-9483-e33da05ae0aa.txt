# Algorithms - Lecture Notes: Binary Search Trees (BST)

## What is a Binary Search Tree?
A BST is a binary tree where left subtree contains only nodes with keys LESS than parent,
and right subtree contains only nodes with keys GREATER than parent.

## BST Search
Start at root. If key < node.key go left, if key > node.key go right.
Time: O(h) where h = height. Worst case O(n), average O(log n).

## BST Insert
Traverse like search until null slot found, insert node there.
Maintains BST property automatically.

## BST Delete - Three Cases
1. Node has no children: just remove it
2. Node has one child: replace node with that child
3. Node has two children: replace with in-order successor (smallest in right subtree)

## Balanced vs Unbalanced
Unbalanced BST degenerates to O(n) for all operations.
AVL trees and Red-Black trees maintain balance automatically, guaranteeing O(log n).

## Practice
Q: For a BST with n nodes in sorted order (worst case), what is search time?
A) O(1) B) O(log n) C) O(n) D) O(n^2)
Answer: C — degenerate BST is a linked list, O(n) search.

## Code Example
```java
// BST Search - recursive
Node search(Node x, Key key) {
    if (x == null) return null;
    int cmp = key.compareTo(x.key);
    if (cmp < 0) return search(x.left, key);
    if (cmp > 0) return search(x.right, key);
    return x;  // found
}
```
