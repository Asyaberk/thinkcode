# Algorithms - Week 3: Merge Sort and Its Analysis

## Learning Objectives
After this lecture, students will understand:
- The divide-and-conquer paradigm
- MergeSort implementation and correctness
- Recurrence relations and Master Theorem
- Bottom-up vs top-down MergeSort

## 1. The Divide-and-Conquer Paradigm
Divide: Split problem into smaller subproblems of the same type.
Conquer: Solve subproblems recursively.
Combine: Merge solutions to produce the final answer.

MergeSort applies this perfectly: divide array in half, sort each half, merge results.

## 2. MergeSort Algorithm
```java
public static void sort(Comparable[] a) {
    Comparable[] aux = new Comparable[a.length];
    sort(a, aux, 0, a.length - 1);
}

private static void sort(Comparable[] a, Comparable[] aux, int lo, int hi) {
    if (hi <= lo) return;
    int mid = lo + (hi - lo) / 2;
    sort(a, aux, lo, mid);      // sort left half
    sort(a, aux, mid+1, hi);    // sort right half
    merge(a, aux, lo, mid, hi); // merge results
}

private static void merge(Comparable[] a, Comparable[] aux, int lo, int mid, int hi) {
    // Copy to auxiliary array
    for (int k = lo; k <= hi; k++) aux[k] = a[k];
    
    int i = lo, j = mid + 1;
    for (int k = lo; k <= hi; k++) {
        if      (i > mid)              a[k] = aux[j++];
        else if (j > hi)               a[k] = aux[i++];
        else if (aux[j].compareTo(aux[i]) < 0) a[k] = aux[j++];
        else                           a[k] = aux[i++];
    }
}
```

## 3. Time Complexity Analysis
Recurrence: T(n) = 2T(n/2) + n
- Divide: O(1) to find midpoint
- Recursive calls: 2 calls on n/2 size
- Merge: O(n) linear scan

By Master Theorem (case 2): T(n) = O(n log n)

This is OPTIMAL for comparison-based sorting — cannot do better.

Best, average, worst case: ALL O(n log n). No bad inputs for MergeSort.

## 4. Space Complexity
MergeSort requires O(n) auxiliary space for the merge step.
In-place MergeSort exists but is complex and has large constant factors.

## 5. Bottom-Up MergeSort (Iterative)
Avoids recursion overhead by merging subarrays of increasing size:
```java
public static void sort(Comparable[] a) {
    int n = a.length;
    Comparable[] aux = new Comparable[n];
    for (int sz = 1; sz < n; sz *= 2)        // sz: subarray size
        for (int lo = 0; lo < n-sz; lo += sz*2)  // lo: subarray index
            merge(a, aux, lo, lo+sz-1, Math.min(lo+sz+sz-1, n-1));
}
```
Same O(n log n) time complexity, O(n) space.

## 6. Key Properties
- Stable: Equal elements maintain relative order (important for multi-key sort)
- Not adaptive: Always O(n log n) regardless of input order
- Not in-place: Requires O(n) extra memory
- Preferred for: Linked lists, external sorting, stability requirement

## Practice Questions
Q1: MergeSort's worst-case time complexity is ___ (blank).
Options: O(n), O(n log n), O(n²), O(log n)
Answer: O(n log n)

Q2: The merge step in MergeSort requires ___ auxiliary space.
Options: O(1), O(log n), O(n), O(n²)  
Answer: O(n)

Q3: Unlike QuickSort, MergeSort is guaranteed ___ in all cases.
Options: O(n log n), O(n), O(1) space, in-place
Answer: O(n log n)

Q4: Bottom-up MergeSort differs from top-down by avoiding ___.
Options: recursion, auxiliary array, the merge step, comparison
Answer: recursion
